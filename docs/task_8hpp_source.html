37  /* HERE BE DRAGONS
38  *
39  * Tasks are handled by an fc::thread . To avoid concurrency issues, fc::thread keeps a reference to the
40  * task in the form of a simple pointer.
41  * At the same time, a task is also a promise that will be fulfilled with the task result, so typically the
42  * creator of the task also keeps a reference to the task (but not necessarily always).
43  *
44  * Because effectively neither fc::thread nor the task creator are responsible for releasing resources
45  * associated with a task, and neither can delete the task without knowing if the other still needs it,
46  * the task object is managed by a shared_ptr.
47  * However, fc::thread doesn't hold a shared_ptr but a native pointer. To work around this, the task can
48  * be made to contain a shared_ptr holding itself (by calling retain()), which happens before the task
49  * is handed to an fc::thread, e. g. in fc::async(). Once the thread has processed the task, it calls
50  * release() which deletes the self-referencing shared_ptr and deletes the task object if it's no longer
51  * in use anywhere.
52  */
